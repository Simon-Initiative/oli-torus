    The required attempt state for page rendering differs between basic and adaptive pages. A basic page needs the "full attempt hierarchy", that is, the resource attempt, and then a map of activity ids on that page to tuples of activity attempt and a part attempt mapping. For example:

    %{
  232 => {%ActivityAttempt{}, %{ "1" => %PartAttempt{}, "2" => %PartAttempt{}}},
  233 => {%ActivityAttempt{}, %{ "1" => %PartAttempt{}, "2" => %PartAttempt{}}}
}

    The adaptive page requires less information, which is also arranged in a different format. It uses simply a mapping of activity resource ids to a small set of data including the attempt guid and the name of the delivery element to use rendering. That looks like:any()

    %{
  232 => %{
    id: 232,
    attemptGuid: 2398298233,
    deliveryElement: "oli-adaptive-delivery"

  },
  233 => %{
    id: 233,
    attemptGuid: 223923892389,
    deliveryElement: "oli-adaptive-delivery"
  }
}
