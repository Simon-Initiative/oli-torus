    number_system_for(locale, system_name)

    View Source

    Specs

    number_system_for(
  Cldr.Locale.locale_name() | Cldr.LanguageTag.t(),
  Cldr.Number.System.system_name()
) :: {:ok, [atom()]} | {:error, {module(), String.t()}}

    Returns the actual number system from a number system type.

    • locale is any valid locale name returned by Cldr.known_locale_names/0 or a Cldr.LanguageTag struct returned by Cldr.Locale.new!/2

    • system_name is any number system name returned by Cldr.known_number_systems/0 or a number system type returned by Cldr.known_number_system_types/0

    This function will decode a number system type into the actual number system. If the number system provided can't be decoded it is returned as is.

    Examples 

    iex> Oli.Cldr.Number.System.number_system_for "th", :latn
{:ok, %{digits: "0123456789", type: :numeric}}

iex> Oli.Cldr.Number.System.number_system_for "en", :default
{:ok, %{digits: "0123456789", type: :numeric}}

iex> Oli.Cldr.Number.System.number_system_for "he", :traditional
{:ok, %{rules: "hebrew", type: :algorithmic}}

iex> Oli.Cldr.Number.System.number_system_for "en", :native
{:ok, %{digits: "0123456789", type: :numeric}}

iex> Oli.Cldr.Number.System.number_system_for "en", :finance
{
  :error,
  {Cldr.UnknownNumberSystemError,
    "The number system :finance is unknown for the locale named \"en\". Valid number systems are %{default: :latn, native: :latn}"}
}

    system_name_from(system_name, locale)

    View Source

    Specs

    system_name_from(
  Cldr.Number.System.system_name(),
  Cldr.Locale.locale_name() | Cldr.LanguageTag.t()
) :: {:ok, atom()} | {:error, {module(), String.t()}}

    Returns a number system name for a given locale and number system reference.

    • system_name is any number system name returned by Oli.Cldr.known_number_systems/0 or a number system type returned by Oli.Cldr.known_number_system_types/0

    • locale is any valid locale name returned by Oli.Cldr.known_locale_names/0 or a Cldr.LanguageTag struct returned by Oli.Cldr.Locale.new!/1

    Number systems can be references in one of two ways:

    • As a number system type such as :default, :native, :traditional and :finance. This allows references to a number system for a locale in a consistent fashion for a given use

    • WIth the number system name directly, such as :latn, :arab or any of the other 70 or so

    This function dereferences the supplied system_name and returns the actual system name.

    Examples 

    ex> Oli.Cldr.Number.System.system_name_from(:default, "en")
{:ok, :latn}

iex> Oli.Cldr.Number.System.system_name_from("latn", "en")
{:ok, :latn}

iex> Oli.Cldr.Number.System.system_name_from(:native, "en")
{:ok, :latn}

iex> Oli.Cldr.Number.System.system_name_from(:nope, "en")
{
  :error,
  {Cldr.UnknownNumberSystemError, "The number system :nope is unknown"}
}

    Note that return value is not guaranteed to be a valid number system for the given locale as demonstrated in the third example.

    to_system(number, system)

    View Source

    Converts a number into the representation of a non-latin number system.

    This function converts numbers to a known number system only, it does not provide number formatting.

    • number is a float, integer or Decimal

    • system_name is any number system name returned by Cldr.known_number_systems/0 or a number system type returned by Cldr.known_number_system_types/0

    There are two types of number systems in CLDR:

    • :numeric in which the number system defines a direct mapping between the latin digits 0..9 into a the number system equivalent. In this case, to_system/2 invokes Cldr.Number.Transliterate.transliterate_digits/3 for the given number.

    • :algorithmic in which the number system does not have the same structure as the :latn number system and therefore the conversion is done algorithmically. For CLDR the algorithm is implemented through Cldr.Rbnf rulesets. These rulesets are considered by CLDR to be less rigorous than the :numeric number systems and caution and testing for a specific use case is recommended.

    Examples 

    iex> Oli.Cldr.Number.System.to_system 123456, :hebr
{:ok, "קכ״ג׳תנ״ו"}

iex> Oli.Cldr.Number.System.to_system 123, :hans
{:ok, "一百二十三"}

iex> Oli.Cldr.Number.System.to_system 123, :hant
{:ok, "一百二十三"}

iex> Oli.Cldr.Number.System.to_system 123, :hansfin
{:ok, "壹佰贰拾叁"}

    to_system!(number, system_name)

    View Source

    Converts a number into the representation of a non-latin number system. Returns a converted string or raises on error.

    • number is a float, integer or Decimal

    • system_name is any number system name returned by Oli.Cldr.known_number_systems/0 or a number system type returned by Oli.Cldr.known_number_system_types/0

    See Oli.Cldr.Number.System.to_system/2 for further information.

    Examples 

    iex> Oli.Cldr.Number.System.to_system! 123, :hans
"一百二十三"

iex> Oli.Cldr.Number.System.to_system! 123, :hant
"一百二十三"

iex> Oli.Cldr.Number.System.to_system! 123, :hansfin
"壹佰贰拾叁"
