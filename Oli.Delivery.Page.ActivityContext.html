    to_thin_context_map(context_map)

    Takes a full context map produced by create_context_map/2 and 'thins' it out to leave only the id, the delivery element name and the attempt guid. This is to allow a page implementation that wishes to avoid injecting a potentially large number of activities (and their full state and model) into a server rendered document.
