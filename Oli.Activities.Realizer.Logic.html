    Oli.Activities.Realizer.Logic (OLI Torus v0.12.7) View Source

    Implements a structured representation of a user specified logic for realizing (aka selecting) activities from the activity bank.

    At an abstract level, the grammar implemented here is: <<Logic>> := <<Clause>> || <<Expression>> <<Clause>> := [<<Clause>>] || [<<Expression>>] <<Expression>> :: Fact Operator Value

    With a <<Clause>> being of type conjunction or disjunction (all vs any)

    The client-side code represents this structure in JSON as:

    {
 conditions: {
    operator: "any",
    children: [{
      operator: "all",
      children: [{
        fact: 'tags',
        operator: 'contains',
        value: [40]
      }, {
        fact: 'objectives',
        operator: 'containsExactly',
        value: [5]
      }]
    }, {
      operator: "all"
      children: [{
        fact: 'tags',
        operator: 'doesNotContain',
        value: [48]
      }, {
        fact: 'text',
        operator: 'contains',
        value: "some text"
      }]
    }]
  }
}
