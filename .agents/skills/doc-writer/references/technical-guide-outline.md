# Torus Technical Guide Outline

## Suggested Sections
1. Purpose and audience
2. System context and architecture
3. Core domains and boundaries
4. Data model and lifecycle
5. Request and event flows
6. Integration points (LTI, LMS, APIs)
7. Operational concerns (jobs, caching, observability)
8. Security and authorization model
9. Testing and verification strategy
10. Troubleshooting and diagnostics
11. Known limitations
12. Related documentation

## Depth Expectations
- Include high-level concepts first, then implementation detail.
- Reference concrete contexts/modules for each domain.
- Capture why design decisions exist, not only what exists.

## Required Callouts
- Multi-tenancy boundaries
- Publication immutability
- Resource/revision versioning behavior
- Role and permission assumptions
